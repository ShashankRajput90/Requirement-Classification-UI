📖 NFR Taxonomy Handbook

Reference guide for Non-Functional Requirement categories.

-- answers appear here --

🔒 Security

Protects system data, encryption, authentication, and secure communication.

  • Examples: "Passwords must be hashed with bcrypt", "All network traffic must use TLS 1.3".
  • Metrics: # vulnerabilities, authentication success rate, encryption coverage.
  • Best Practices: Use least privilege, regular security audits, follow OWASP guidelines.

⚡ Performance

Defines response time, throughput, and system speed.

  • Examples: "System must respond within 2s", "Process 1000 transactions/sec".
  • Metrics: Response time, throughput, latency.
  • Best Practices: Optimize algorithms, load testing, caching frequently accessed data.

👤 Usability

Ease of use, accessibility, and intuitive user interfaces.

  • Examples: "UI must comply with WCAG 2.1 AA", "Checkout under 3 steps".
  • Metrics: User satisfaction score, error rates, completion time.
  • Best Practices: Conduct usability testing, use consistent design patterns, provide clear guidance.

📈 Reliability

Ensures system stability, uptime, and fault tolerance.

  • Examples: "System uptime ≥ 99.9%", "Automatic failover on server crash".
  • Metrics: MTBF, downtime, error rates.
  • Best Practices: Implement monitoring, redundancy, and backup strategies.

🛠 Maintainability

Ease of debugging, updating, and extending the system.

  • Examples: "Modules must be loosely coupled", "Code must follow style guides".
  • Metrics: Mean time to repair, code complexity, documentation coverage.
  • Best Practices: Use modular design, maintain tests, keep documentation updated.
